## 1. OpenProse

**Money.** OpenProse says it raised **$1.25 million** from Jonathan Abrams's 8-Bit Capital, Otis Chandler, and angels, and reports more than **7,000 installs**. Its first disclosed customer is a medical research lab starting with a **$10,000 pilot** intended to convert into a $10,000-per-month contract. The public repository showed about **1,500 stars, 121 forks, and 247 commits** when accessed. These are unusually concrete early signals, although the contract conversion and retention are not yet verified. [YC profile, accessed June 21, 2026](https://www.ycombinator.com/companies/openprose) [GitHub, accessed June 21, 2026](https://github.com/openprose/prose)

**Technology — disclosed.** OpenProse is a declarative language for agent work. Developers write versioned Markdown contracts describing required inputs, maintained outputs, tools, and continuity rules. The Forme layer resolves dependencies into a graph; the Reactor runtime watches declared inputs, reruns only nodes whose fingerprints changed, validates postconditions, and stores content-addressed receipts. A deterministic reconciler—not an LLM judge—decides whether work should wake or be skipped. The project ships offline replay datasets, a zero-runtime-dependency core, an SDK, CLI, devtools, and adapters for live model execution. [Technical site, accessed June 21, 2026](https://prose.md/) [Documentation, accessed June 21, 2026](https://docs.prose.md/)

**Technology — assessment.** The architecture applies Terraform/Kubernetes-style desired-state reconciliation to long-running model work. That is stronger than a prompt-chain DSL because unchanged inputs can reuse a prior result at zero inference cost, failed postconditions preserve the last valid state, and receipts make execution inspectable. The difficult engineering is defining canonical fingerprints that ignore irrelevant changes without hiding meaningful drift; handling nondeterministic model outputs; preventing stale memoized truth; and keeping DAG execution recoverable across crashes, retries, and partial side effects. External actions still require idempotency and compensating operations—the receipt ledger cannot undo a sent email or database mutation.

The repository openly identifies important gaps. Current receipts are chain-consistent but do not yet cryptographically bind every world-model artifact, and they lack actor and timestamp fields. Benchmarks remain pending. Commodity pieces include Markdown parsing, DAG schedulers, caches, and agent SDKs. The potential moat is the language contract, portable ecosystem, deterministic runtime semantics, replay corpus, and developer adoption. Competition includes Temporal, Dagster, Prefect, LangGraph, PydanticAI, workflow engines, and custom agent harnesses.

**Founders.** Dan Barrett has 12 years of software experience, five years building agents, and prior founder experience. The disclosed team now includes two senior additions, including ex-Google and agent-harness experience, but YC publicly profiles only Barrett. Founder-market fit is strong because the product is an inspectable systems artifact rather than a marketing shell. The gap is enterprise runtime operations and security support if the project becomes responsible for persistent production workflows.

**Verdict — 8.3/10, High confidence.** OpenProse has the best combination of inspectable architecture, honest limitations, open-source pull, and early paid demand. The main risk is whether a novel language becomes a standard or remains a useful niche layered over larger workflow platforms.

## 2. Tolmo

**Money.** Tolmo is a 12-person YC Spring 2026 seed company with YC's standard financing and no separately disclosed round. It names no paying customer, revenue, deployment count, or retention metric. Commercial evidence is therefore weak; the stronger signal is a team built from Sqreen, Datadog, Apple security, and Node.js security work. [YC profile, accessed June 21, 2026](https://www.ycombinator.com/companies/tolmo)

**Technology — disclosed.** Tolmo ingests read-only metadata from code, CI, cloud services, containers, databases, identity, observability, and security tools into a continuously updated production knowledge graph. Specialized agents perform asset discovery, adversarial pentesting, impact verification, triage, remediation, and pull-request review. Its screenshots show graphs with tens of thousands of nodes and edges. The CLI supports SQL and Cypher queries, finding management, authenticated proxying to GitHub, AWS, Linear, Sentry, and Datadog, CI integration, threat models, monitors, and a time-history view. Credentials remain in Tolmo's backend rather than entering the local CLI. [Platform, accessed June 21, 2026](https://tolmo.com/) [CLI documentation, accessed June 21, 2026](https://docs.tolmo.com/)

**Technology — assessment.** The graph is the core. A credible schema must connect repositories, commits, images, workloads, identities, secrets, data stores, routes, alerts, and historical configuration. Pentesting agents can then rank attack paths using reachability and privilege relationships instead of treating scanner findings independently. Remediation must recalculate blast radius, propose a bounded change, run tests, and verify that the path is actually closed.

The hardest work is keeping the graph correct under constant infrastructure change. Cloud APIs are eventually consistent; tags and service ownership are incomplete; runtime dependencies differ from declared configuration; and third-party integrations break. Autonomous exploitation also creates liability. Tests need strict scope, rate limits, data-handling rules, isolation, and proof that a successful exploit did not damage production. Read-only ingestion reduces deployment risk but conflicts with autonomous remediation unless write credentials are later introduced through a carefully separated control path.

Knowledge graphs, cloud connectors, LLM agents, scanners, and ticket creation are individually commodity. Differentiation could come from historical graph quality, verified exploit paths, cross-vendor context, and closed-loop fixes. Wiz, Orca, Palo Alto, CrowdStrike, Datadog, Snyk, Semgrep, Endor Labs, and numerous agentic-security startups can attack adjacent layers.

**Founders.** Pierre Betouin led security products at Datadog after Sqreen's acquisition and previously spent nine years at Apple conducting security assessments. Vladimir de Turckheim was an early Sqreen engineer, Datadog staff engineer, Node.js core collaborator, and founder of the Node.js security program. That is exceptional technical and distribution fit. The gap is not security expertise; it is proving that autonomous actions outperform trusted incumbent workflows without increasing operational risk.

**Verdict — 8.1/10, Medium-high confidence.** The architecture and founders are strong enough to underwrite despite absent traction. The score will rise only with named production deployments, measured exploit verification precision, remediation acceptance rates, and evidence that the graph stays accurate at scale.

## 3. General Instinct

**Money.** General Instinct is a two-founder YC Spring 2026 seed company with no outside round disclosed. It reports one production example: a multimodal classifier on a Jetson Orin NX with a 111-millisecond cold start, every decision inside a 150-millisecond latency budget, and no cloud calls. The customer, dataset, baseline model, accuracy retention, power draw, and sustained throughput are undisclosed, so this validates feasibility more than demand. [YC profile, accessed June 21, 2026](https://www.ycombinator.com/companies/general-instinct) [Launch YC, May 2026](https://www.ycombinator.com/launches/QPE-general-instinct-the-deployment-layer-for-physical-intelligence)

**Technology — disclosed.** Instinct Edge accepts a model, target hardware, and latency budget, then returns an offline runtime for Jetson devices, mobile NPUs, ARM CPUs, Apple Neural Engine, and Snapdragon systems. The company says its pipeline combines model distillation, quantization, compression recipes, custom CUDA, Metal, and ARM NEON kernels, plus continuous data-to-serving tooling.

**Technology — assessment.** The likely compiler pipeline profiles the source graph, identifies unsupported or expensive operators, applies pruning or quantization-aware transformations, chooses target-specific kernels, calibrates activation ranges, and benchmarks candidate builds against latency, memory, power, and accuracy constraints. Distillation can recover quality lost through smaller architectures or lower precision. Runtime packaging then has to manage device-specific memory, sensor preprocessing, batching, thermal throttling, and model updates.

The difficult part is multi-objective optimization. A model can meet median latency while failing p99 under heat, memory pressure, camera bursts, or concurrent workloads. Quantization may damage rare-class recall, depth estimation, or control stability even when aggregate accuracy looks stable. Physical systems also need deterministic fallback behavior when the model is uncertain. Deployment across NVIDIA, Apple, Qualcomm, and generic ARM creates a large compatibility matrix with different compilers, operator support, numerical behavior, and debugging tools.

ONNX, TensorRT, Core ML, TFLite, TVM, ExecuTorch, vendor SDKs, and quantization libraries are commodity alternatives. The moat would be automated search across compression/runtime choices, proprietary kernel work, a benchmark corpus of real edge workloads, and production feedback showing accuracy under field conditions—not a no-code wrapper.

**Founders.** Bill Jiao worked on Siemens's first multimodal foundation model and holds two protein-related patents. Guanming Wang has Google DeepMind and Tsinghua research experience and an ICRA publication history. The pair has direct model and systems credibility. Missing capabilities include safety certification, embedded fleet operations, and sales into robotics programs with long validation cycles.

**Verdict — 8.0/10, Medium-high confidence.** Edge deployment is a hard, measurable bottleneck and the disclosed stack reaches below generic MLOps. The central question is whether General Instinct can preserve task quality and reliability across a wide hardware/model matrix better than vendors and internal performance teams.

## 4. shotwell.ai

**Money.** shotwell.ai is a four-founder YC Spring 2026 seed company with no disclosed outside round or revenue. Its website shows **Ultra** and **Parametric** under “Trusted by,” but does not state whether these are paid customers, pilots, design partners, or data contributors. No processed-video hours, annotation price, accuracy, or retention is public. [YC profile, accessed June 21, 2026](https://www.ycombinator.com/companies/shotwellai) [Company site, accessed June 21, 2026](https://shotwell.ai/)

**Technology — disclosed.** Robotics companies send Shotwell training or deployment video. Its models examine frames, segment long recordings into discrete actions, label each action against a task definition or standard operating procedure, and detect failures. The company promises dense annotation with near-instant turnaround and exposes a demo using open-source robot data.

**Technology — assessment.** The likely system combines video encoders, temporal action segmentation, vision-language models, and task-specific rubrics. A long episode is first divided into state transitions or action boundaries, then each segment is labeled with objects, contact events, motion, task stage, and outcome. Deployment monitoring compares observed sequences with an expected plan and flags omissions, loops, unsafe motion, or terminal failure.

This is materially harder than ordinary image labeling. Robot failures can be subtle: a grasp is unstable but not yet dropped, a drawer is almost closed, force is excessive, or the task succeeds through an unsafe path. Camera viewpoint, occlusion, lighting, embodiment, and task vocabulary vary. Near-instant annotation may use large multimodal models, but cost and latency can become severe on frame-dense data. A practical cascade would use cheap embeddings and change detection to reduce frames, then stronger models on candidate boundaries, with human review for low-confidence labels.

Commodity video models and annotation tools are abundant. Shotwell's moat would be a cross-robot ontology, temporal failure datasets, uncertainty calibration, fast task adaptation, and the loop from deployment failures back into training examples. Competition includes Scale AI, V7, Encord, Roboflow, internal labeling teams, and robotics-specific data platforms.

**Founders.** Farhan Khan worked on humanoids at Sunday Robotics and autonomous vehicles at Tesla. Ali Abdalla and Ali Abid co-founded Gradio, acquired by Hugging Face and used by a large developer base. Nour Eldifrawy brings integration experience from Merge.dev and Roblox. The team has rare robotics, ML tooling, and distribution overlap. It still needs proof of annotation science and enterprise data handling.

**Verdict — 7.9/10, Medium confidence.** Robotics data observability is a real infrastructure category and the team is unusually strong. Confidence is capped until Shotwell publishes agreement with expert annotators, failure recall, cost per video hour, supported task families, and customer deployment evidence.

## 5. AquaShield

**Money.** AquaShield has YC's standard seed financing and no disclosed outside round. The founders say they began with MIT's facilities team and are now working with one of the world's largest residential portfolios. Neither organization, building count, paid status, prevented-loss evidence, nor sensor fleet size is disclosed. The company claims localization can turn a $100,000-plus remediation into a sub-$2,000 targeted repair; that is a scenario claim, not audited savings. [YC profile, accessed June 21, 2026](https://www.ycombinator.com/companies/aquashield) [Launch YC, May 2026](https://www.ycombinator.com/launches/QRA-aquashield-ai-protection-against-water-damage-in-buildings)

**Technology — disclosed.** Non-invasive sensors install on existing pipes without cutting or downtime. AquaShield learns normal building flow patterns, detects anomalies, and claims to localize a leak to a pipe segment despite noisy consumption.

**Technology — assessment.** The sensors could use acoustic, vibration, pressure, ultrasonic, or clamp-on flow measurements. Multiple synchronized nodes create a graph of the water network. Models learn recurring demand signatures and identify deviations in timing, frequency, pressure, or acoustic propagation. Localization likely combines sensor geometry, event timing, attenuation, and a building-specific pipe topology.

The hard problem is separating leaks from legitimate irregular use: cleaning, irrigation, appliance cycles, tenant behavior, pressure transients, maintenance, and seasonal occupancy. A model trained on one building may not transfer across pipe materials, diameters, pumps, valves, and construction layouts. Battery life, wireless coverage through concrete, sensor attachment, calibration drift, and false-alarm service cost determine unit economics. The system also needs a reliable way to map a statistical location to an actionable physical pipe segment when drawings are incomplete.

Commodity leak sensors detect local moisture or total-flow anomalies. AquaShield's stronger claim is portfolio-scale, non-invasive localization. Defensibility would come from multi-building labeled leak data, network inference, deployment playbooks, and insurer or property-manager distribution. Flo by Moen, Phyn, WINT, Alert Labs, Eddy Solutions, and building-management vendors compete.

**Founders.** Paul Beckers and Marguerite Benoist previously conducted robotics research at MIT and Harvard, respectively. That background fits sensing, signal processing, and deployment under uncertainty. Public profiles do not show plumbing, facilities operations, insurance, or large-scale hardware manufacturing experience, which are material commercial gaps.

**Verdict — 7.8/10, Medium confidence.** The product links technically difficult sensing to a clear loss budget. The investment case depends on measured localization error, false alarms per building-month, sensor maintenance, prevented claims, and repeatable installation economics.

## 6. BentoLabs AI

**Money.** BentoLabs is a five-person YC Spring 2026 seed company with no disclosed outside financing. It says it is working with “teams at unicorn scale,” but names no customer, contract, revenue, trace volume, or retention. Internal experiments report Terminal-Bench 2.0 pass@1 increasing from 42.2% to 52.4% with the same model, tools, and budget, plus a 2.6-times score increase on ARC-AGI-3. These results are not independently reproduced. [YC profile, accessed June 21, 2026](https://www.ycombinator.com/companies/bentolabs-ai) [Launch YC, May 31, 2026](https://www.ycombinator.com/launches/Qcw-bentolabs-ai-monitoring-and-learning-layer-for-long-running-agents)

**Technology — disclosed.** Bento ingests OpenTelemetry-native spans across agent frameworks, trains plain-English regression signals on a customer's traces, backfills history, groups alerts into incidents, detects behavioral drift, and traces failures to individual calls. Its learning layer converts production fixes into versioned artifacts—skills, subagents, tools, and a written “Book”—then evaluates changes offline, in CI, and on canary traffic. Prompts, tools, skills, and models are versioned and reversible. [Product site, accessed June 21, 2026](https://bentolabs.ai/)

**Technology — assessment.** Monitoring is plausible: traces contain prompts, outputs, tool calls, latency, errors, user feedback, and downstream outcomes. A regression signal can be implemented as rules, classifiers, LLM judges, or outcome-linked detectors. The harder claim is recursive learning. The system must identify the causal failure, propose a fix, evaluate it on representative counterexamples, prevent overfitting, and promote it without degrading unrelated behavior.

Production traces are biased toward observed users and may contain secrets or personal data. LLM-based detectors can be expensive and inconsistent. Behavioral drift may be benign, while an apparently successful fix can hide a new failure mode. Safe promotion needs holdouts, shadow execution, confidence intervals, rollback, and human approval for material changes. Commodity observability and eval platforms include LangSmith, Arize, Braintrust, Langfuse, Datadog, Weights & Biases, and Helicone. Bento's moat would require superior failure clustering, outcome labels, and evidence that its generated artifacts improve production metrics rather than benchmarks alone.

**Founders.** Abhinav Soni and Kaushik ASP were the first two hires at Emergent, where they say they built production coding-agent infrastructure used by five million users and helped scale the company rapidly. That is direct founder-market fit. The material gap is independent validation of both the prior scale claims and Bento's benchmark methodology.

**Verdict — 7.7/10, Medium confidence.** Bento correctly connects observability, evaluation, versioning, and rollback. The score is capped because “self-learning” is where evaluation leakage and silent regressions become most dangerous; production outcome evidence matters more than internal benchmark lift.

## 7. Allowance

**Money.** Allowance is a single-founder YC Spring 2026 seed company with no additional round disclosed. The iPhone app and desktop-agent setup are live in early public beta. The founder reports using it for real food, reservation, and ecommerce transactions, but the company provides no active-user count, payment volume, take rate, issuer relationship, fraud loss, approval rate, or revenue. [YC profile, accessed June 21, 2026](https://www.ycombinator.com/companies/allowance) [Product site, accessed June 21, 2026](https://useallowance.com/)

**Technology — disclosed.** A user approves a purchase on an iPhone, setting amount, merchant, and expiration constraints. Allowance generates a temporary payment credential so the agent never receives the underlying card number. It supports one-time, scheduled, and recurring transactions; logs receipts; and allows instant revocation. The product is designed to work inside existing agent environments.

**Technology — assessment.** The clean architecture separates intent, authorization, credential issuance, transaction enforcement, and reconciliation. An agent requests a payment capability with structured merchant and amount fields. The user approves a signed policy. Allowance then provisions a network token or virtual card credential whose authorization controls enforce limits. A webhook or ledger matches authorization, clearing, refund, and dispute events back to the task.

The difficult edge cases are payment semantics. Merchant locks can fail when processors use unexpected descriptors; hotel and rental deposits exceed initial estimates; tips and currency conversion change totals; subscriptions recur; partial captures and refunds arrive later; and agents can be socially engineered into buying the wrong item from an allowed merchant. Strong controls require merchant-category and descriptor normalization, step-up approval, device authentication, velocity rules, fraud detection, and clear dispute responsibility. PCI scope, card-network rules, money-transmission analysis, issuer sponsorship, chargebacks, and customer support may dominate software development.

Virtual-card issuance and approval UX are available from Stripe Issuing, Marqeta, Lithic, privacy-card products, and banks. Differentiation could come from agent-native authorization semantics, cross-agent integrations, reputation, and transaction-risk data.

**Founders.** Dasmer Singh previously led product for Cash App Families, was an early iOS engineer at Venmo, and worked at Uber and Petal. That is excellent consumer-payments and controls experience. Single-founder engineering, compliance, fraud, and support bandwidth is the major execution risk.

**Verdict — 7.6/10, Medium confidence.** Allowance addresses a real missing capability with the correct least-privilege pattern. The moat and economics remain unproven until it discloses issuer infrastructure, approval-to-completion rates, payment volume, fraud/chargeback loss, and agent distribution.

## 8. Twolabs

**Money.** Twolabs has YC's standard seed financing and no disclosed outside round, customer, pilot, revenue, robot sale, or facility agreement. The team says the Tobi prototype design is complete and its AI stack is in active development. This is pre-deployment evidence. [YC profile, accessed June 21, 2026](https://www.ycombinator.com/companies/twolabs) [Launch YC, May 28, 2026](https://www.ycombinator.com/launches/QWa-twolabs-humanoid-robots-for-caregiving)

**Technology — disclosed.** Twolabs is building a modular semi-humanoid platform for nursing homes and senior living. Intended tasks include feeding, dressing, medication reminders, object retrieval, walking stabilization, wayfinding, and companionship. The company says it is training vision-language-action models and building hardware and social intelligence in-house.

**Technology — assessment.** Caregiving combines manipulation, mobility, conversation, and safety in close human contact. The robot needs perception of people, objects, pose, and intent; compliant force-controlled arms; fall-safe mobility; speech and identity systems; task planning; and immediate human takeover. VLA policies may map vision and language instructions to actions, but real facilities require deterministic safety envelopes, force/torque limits, collision avoidance, uncertainty detection, and logs suitable for incident review.

Feeding and dressing are harder than warehouse picking because objects deform, people move unpredictably, and errors can injure vulnerable users. Walking support raises medical-device and liability questions. Social intelligence adds consent, privacy, emotional dependency, and hallucination risk. Data collection is constrained because resident video and health context are sensitive. Unit cost includes actuators, batteries, sensors, maintenance, teleoperation, insurance, facility integration, and staff training. If humans intervene frequently, the robot may not beat caregiver economics.

Robot components, foundation models, and teleoperation stacks are increasingly accessible, but full-system reliability is not. The moat could become proprietary caregiving demonstrations, safe manipulation policies, facility workflow integration, and a service network. Competitors include 1X, Figure, Apptronik, Tesla Optimus, Diligent Robotics, Labrador Systems, and specialized assistive devices.

**Founders.** Sardor Rahmatulloev worked on model training and GPU optimization at Meta Superintelligence Labs and led a large Cornell combat-robotics team. Danyal Ahmad has an AI master's from Georgia Tech, Salesforce engineering experience, and video-model work. Their personal motivation and technical fit are credible; neither profile shows medical-device regulation, elder-care operations, hardware manufacturing, or field service.

**Verdict — 7.3/10, Medium-low confidence.** The problem is important and the team can build a serious prototype. The score reflects extreme pre-product risk: Twolabs needs a functioning robot, bounded initial task, facility partner, intervention-rate data, and a safety/regulatory plan.

## 9. Zenbu

**Money.** Zenbu is a single-founder YC Spring 2026 seed company with no disclosed outside round, revenue, active-user count, paid team, or retention. A macOS beta is publicly downloadable. Founder distribution is stronger than product traction: Rob Pruzan co-authored React Scan, which YC says has 22,000 GitHub stars and 800,000 weekly installs. [YC profile, accessed June 21, 2026](https://www.ycombinator.com/companies/zenbu-2) [Launch YC, June 7, 2026](https://www.ycombinator.com/launches/Qey-zenbu-the-extensible-ide-for-managing-coding-agents)

**Technology — disclosed.** Zenbu is a hackable desktop interface built around the Pi coding agent. It supports multiple model providers, Pi commands and extensions, parallel agents in Git worktrees, side-by-side sessions, context switching, and pull-request creation. Zenbu.js provides a plugin layer, and the built-in agent can modify the interface by creating plugins. [Product site, accessed June 21, 2026](https://tryzenbu.dev/)

**Technology — assessment.** The useful system boundary is not chat; it is workspace isolation and review. Each task should receive a branch or worktree, sandboxed process environment, explicit tool permissions, and a record of commands and edits. The IDE then needs to detect conflicting changes, summarize diffs, run tests, compare task intent with output, and merge or discard work safely. Plugins require capability controls because an extension can access code, credentials, terminals, and model context.

Parallelism creates orchestration costs. Multiple agents may edit shared files, duplicate work, build against stale branches, or pass tests individually but fail when merged. The product's defensibility depends on reliable conflict handling, verification, extension quality, and workflow data—not the number of visible chat panes. VS Code extensions, Cursor, Codex, Claude Code, Windsurf, Conductor, and open-source agent managers can reproduce much of the interface.

**Founders.** Pruzan worked on Next.js at Vercel and built an internal development environment in addition to React Scan. That is strong developer-tool credibility and an existing open-source audience. Single-founder support, security, cross-platform delivery, and rapid competition are the gaps.

**Verdict — 7.1/10, Medium confidence.** Zenbu has a clear extensibility thesis and credible founder distribution. It must prove that plugins and worktree orchestration improve merged-code quality and throughput rather than adding another interface over interchangeable coding agents.

## 10. Wato

**Money.** Wato is a two-founder YC Spring 2026 seed company with no outside round, named customer, user count, revenue, or deployment evidence. The product remains access-request based. Its website claims more than 400 connectors, but does not identify whether these are native, brokered through an integration provider, or production-tested. [YC profile, accessed June 21, 2026](https://www.ycombinator.com/companies/wato) [Launch YC, June 7, 2026](https://www.ycombinator.com/launches/Qet-wato-a-shared-ai-workspace-for-teams)

**Technology — disclosed.** Wato exposes one Model Context Protocol endpoint through which Codex, Claude, Cursor, and other agents can access versioned company memory, reusable skills, approved tools, shared cloud sessions, scheduled automations, live artifacts, and traces. Permissions can be scoped by organization, team, user, connector, and tool. Cloud sessions include a desktop, filesystem, browser, environment, Git, and computer use. [Product site, accessed June 21, 2026](https://www.watolabs.com/)

**Technology — assessment.** Wato is effectively an enterprise control plane for agent context and execution. Authentication maps an agent session to a user and organization; policy filters tools and data; retrieval assembles relevant memories and skills; a sandbox executes actions; and traces record access and outputs. Versioning is essential because company memory changes, corrections must propagate, and agents need provenance for every reused fact.

The hardest problem is authorization across heterogeneous systems. An employee may read a Slack channel but not export it into a reusable team memory. A sales agent may query Salesforce without seeing engineering incidents. Revoking source access should also revoke derived artifacts where required. Prompt injection through connected documents can turn broad MCP access into a confused-deputy attack. Cloud sessions require tenant isolation, secret management, network controls, signed images, audit logs, and limits on what computer-use agents can transmit.

Memory, connectors, sandboxes, workflow scheduling, and MCP gateways are commodity pieces. The moat would need to emerge from policy semantics, provenance, administration, trusted connector quality, and organization-wide workflow adoption. Microsoft, Google, OpenAI, Anthropic, Glean, Dust, Moveworks, and many agent platforms can bundle adjacent capabilities.

**Founders.** Arihan Varanasi and Rahul Rejeev studied computer science at Stanford, but public profiles disclose little production infrastructure, enterprise security, or prior company-building experience. They understand the current MCP adoption problem, yet the product scope is unusually broad for a two-person team.

**Verdict — 6.9/10, Medium-low confidence.** Wato targets a real governance gap and presents a coherent product surface. The score is limited by absent customer proof, broad scope, thin founder evidence, and platform risk from model vendors that can integrate shared memory and permissions directly.
